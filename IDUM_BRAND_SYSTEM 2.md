# IDUM Brand Design System

Version: 1.0\
Purpose: Provide a consistent visual design system for IDUM website,
reports, and digital products.

This document defines the **visual identity rules** for the organization
so that future UI development remains consistent.

------------------------------------------------------------------------

# Brand Principles

The visual identity should communicate:

-   Scientific credibility
-   Global cooperation
-   Environmental stewardship
-   Institutional authority
-   Calm leadership

The organization should visually feel similar to:

-   IPCC
-   UN Environment Programme
-   Ocean Cleanup
-   Nature Conservancy

Avoid visual styles that feel:

-   playful
-   startup-like
-   activist
-   cartoonish

The tone should feel **institutional, credible, and scientific**.

------------------------------------------------------------------------

Body

-   18px\

Small

------------------------------------------------------------------------

## Typography Rules

-   Body text should never be below **16px**
-   Maximum text width should be **640px**
-   Headings should be bold but not heavy
-   Avoid light font weights

------------------------------------------------------------------------

# Corner Radius

UI elements should feel precise and technical.

border-radius: 6px

Avoid:

-   pill shapes
-   overly rounded UI

------------------------------------------------------------------------

# Buttons

Padding:

14px 20px

Border radius:

6px

------------------------------------------------------------------------

# Icons

Recommended icon library:

Lucide Icons

Icon rules:

-   style: outline
-   stroke width: 1.5
-   sizes: 20px / 24px

Icons should represent:

-   science
-   research
-   ocean
-   collaboration
-   governance
-   data

Avoid:

-   cartoon animals
-   playful environmental graphics

------------------------------------------------------------------------

# Photography Direction

Images should emphasize:

-   ocean scale
-   research activity
-   scientific mapping
-   environmental monitoring
-   international cooperation

Examples of good imagery:

-   aerial ocean imagery
-   research vessels
-   sonar mapping
-   scientists working
-   underwater mapping

Avoid:

-   generic beach tourism imagery
-   volunteers cleaning beaches
-   stock environmental photos

The issue is **serious, hidden, and global**.

------------------------------------------------------------------------

# Illustration Style

If illustrations are used they should resemble:

-   scientific diagrams
-   bathymetric maps
-   sonar visualizations
-   ocean cross-section graphics

Avoid cartoon-style environmental illustrations.

------------------------------------------------------------------------

# Data Visualization

Charts and maps should use the following palette:

-   Deep Ocean Blue
-   Sea Green
-   Slate Gray
-   Light Ocean Tint

Data visualization should feel similar to:

-   UN reports
-   IPCC publications
-   scientific dashboards

------------------------------------------------------------------------

# Motion

Motion should be subtle and purposeful.

Use motion for:

-   map reveals
-   section transitions
-   hover states

Avoid:

-   playful animation
-   bouncing elements
-   excessive motion

------------------------------------------------------------------------

# Visual Tone Summary

The design language should feel:

-   credible
-   scientific
-   global
-   institutional
-   environmentally responsible

IDUM should visually communicate the role of a **global coordinating
authority on underwater munitions**, similar to how IPCC operates for
climate science.

------------------------------------------------------------------------

# Implementation Guidance

When designing UI components or web pages:

1.  Use the defined typography scale.
2.  Use ocean blues as primary UI colors.
3.  Maintain a calm institutional layout.
4.  Prefer photography over icons.
5.  Keep UI minimal and serious.

This system should guide:

-   website design
-   presentations
-   reports
-   graphics
-   dashboards
-   visual communications.
